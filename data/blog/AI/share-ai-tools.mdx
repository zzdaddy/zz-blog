# Kimi

Kimi是月之暗面科技有限公司（Moonshot AI）开发的AI助手。

月之暗面，这名字起的真好，有点中二热血的感觉，也有点诗意。

有三个显著特点：

1. 可解析网站地址的内容
2. 可上传文档解析
3. 超长上下文

可解析网站地址现在有很多AI都能做到了，但是Kimi是我一直在用的，也没有换。

这东西就像是理发店一样，除非哪天托尼老师发挥失常，给你整了个大无语造型，否则是轻易不会换的

![](https://img.zzstudio.cn/%E7%A6%81%E6%AD%A2%E6%AE%B4%E6%89%93%E9%A1%BE%E5%AE%A2.jpeg)

可上传文档解析，这个绝对也是工作利器，方便把优秀的文案之类的，拿来学习学习😛

这两个功能都是一样的用途：初加工文案

对于勤快的人来说，大大加快了处理速度，提升效率

对于懒人来说，相当于以前用人给做了饭，但需要去餐桌上吃，现在是直接躺在床上等着喂饭，爽的一批。

并且现在很多副业、自媒体，都免不了处理文字，有了AI辅助之后才是真的起飞。

不然你像我靠自己码字，除了素材要多，脑洞还得大。

不光要码字，还要敲代码，也就是我的头发长得牢靠，不然早就霍霍没了。

通过Kimi概括出大纲或者精华之后，往往都需要二次加工

所以怎么使用关键字很关键，这点和下边的coze是一样的，我放在下边说

这就是你的不对了

你想让一个人或者工具帮你做一个事儿的时候

你**首先要知道这个人或者工具，有多大的能力。 **

打个比方，领导要分配一个项目的工作，咱首先得确定同学们大概能承受多大的工作量，因为能力有高低，效率有快慢，你给一个效率慢同学分配更多的活和更短的时间，那显然是不合理。

AI也是一样，虽然他们知道的多，但是就是因为信息太多了，你给的要求太宽泛，AI就很难答出你心意的答案，**他会尽量把他知道的都告诉你**，一旦多了，就不是我们想要的了，因为我们问问题就是为了得到精准的回复。

所以**其次就是，要准确的精细的描述出自己的需求。**

比如你要写一篇文章，那你肯定是希望AI是个一个资深的作家，因为你想让他帮你分析文章，或者帮你扩写，帮你润色。 

所以要先给AI设定一个**背景**

然后要想把一个想写文章这样的流程比较长的活，让一个不太好把控的AI完成

咱就得让他分段进行

类似项目需求分配，把活分给下边的同学A。首先咱自己作为掌控者，要熟悉需求，然后把需求拆解，分成123三个阶段，并且为了项目稳定进行，还得让A同学完成一个阶段就汇报一次，如果不合适他就要再改。这样才能得到咱们想要的结果

所以AI也是一样的

AI可以理解自然语言，所以你可以像写代码一样去给他分配

设定完\<**背景**>后，先定义一系列\<**任务**>，每个任务里，把你要求的1234都列出来，并且要求他每一条任务完成后，要询问你是否合适后再进行下一条任务

你看，这完全就和写了一段代码逻辑一样

```
// 伪代码
class
// 定义基础的属性
 constructor() {
	 你是一个作家
 }

// 定义他的各项任务
fn 任务1
	if (询问是否合格) return 执行
	任务1（） 

fn 任务2
	...

fn 任务3
	...

// 每次发送消息都是调用run
run() {
	if(得到了任务1指令) {
		任务1（）
	}
	if (...) {
		...
	}

}
```
